| 시스템 | 역할 | 결정권 |

| **Orchestrator v2.0** | 의도 분석, 노드 매핑, Causality 구축, 상태 전환 총괄 | REALIZED 최종 확정 |

## 검증 3대 원칙 — 적용 기준

### 원칙 1. 계보적 불변성 (Lineage Immutability)

**판단 질문:** "부모 노드의 기하학적 형태(매스, 인층, 경계)가 자식 결과물에서 1:1로 유지되는가?"

- 부모 아트보드에서 규정한 건물 매스, 층수, 대지 경계는 하위 프로세스에서 변경 불가
- 시점이 바뀌거나(N4), 도면화되거나(N2), 이미지로 실체화되어도(N3) 기하학적 골격은 동일해야 함
- 위반 예시: 부모 스케치의 3층 매스가 자식 이미지에서 5층으로 렌더링되는 경우

**주로 적용되는 노드:** N2(Plan), N3(Image), N4(Viewpoint)

---

### 원칙 2. 결정론적 하드데이터 (Deterministic Hard-Data)

**판단 질문:** "결과물이 측정 가능한 물리 법칙과 규격 데이터(AEPLSchema)에만 근거하는가? AI 주관 추론(할루시네이션)이 없는가?"

- GFX 100S 광학 파라미터(초점거리, 조리개, 센서 특성) 기반 렌더링 여부
- AEPLSchema v4 수치(폭, 높이, 깊이, 공극비, 재료 규격) 준수 여부
- 위반 예시: 치수 데이터 없이 "자연스러운 비례감"으로 입면 수치를 추정하는 경우

**주로 적용되는 노드:** N3(Image), N5(Elevation)

---

### 원칙 3. 기능적 해결성 (Functional JTBD)

**판단 질문:** "해당 노드만이 수행할 수 있는 고유한 설계 로직이 실행되어 데이터 밀도가 유의미하게 증가했는가?"

- 각 노드는 VOID 상태의 아트보드에 고유한 Jobs-to-be-Done을 주입해야 함
- 단순 렌더링이나 형태 복사는 JTBD 실행으로 인정하지 않음
- 위반 예시: N6(Diagram)에서 공간 로직 분석 없이 이미지를 그대로 다이어그램 배경으로 사용하는 경우

**주로 적용되는 노드:** N1(Planners), N6(Diagram), N7(Print)

## 노드별 Cut-off 및 판단 중점

### N1 — Planners (Cut-off: 1)

| 원칙 | 적용 여부 | 판단 기준 |
|------|-----------|-----------|
| 계보적 불변성 | 해당 없음 | Planners는 최상위 노드 — 부모 없음 |
| 결정론적 하드데이터 | 보조 | 법규, 용적률 등 정량 데이터 기반 여부 |
| **기능적 해결성** | **핵심** | 전략/리스크/JTBD 실행 로드맵이 구체적으로 산출되었는가 |

**Cut-off 1의 이유:** 전략 기획 노드로 단일 실행으로 충분한 데이터 밀도 확보 가능. 반복 검증의 실익이 낮음.

---

### N2 — Plan (Cut-off: 1)

| 원칙 | 적용 여부 | 판단 기준 |
|------|-----------|-----------|
| **계보적 불변성** | **핵심** | 스케치 원본의 구조적 형태가 도면에 그대로 반영되었는가 |
| 결정론적 하드데이터 | 보조 | 모듈 그리드, 치수 체계 준수 여부 |
| 기능적 해결성 | 보조 | 도면 지역화(공간 레이블, 실명 배정) 수행 여부 |

**Cut-off 1의 이유:** 스케치 → 도면 변환은 단일 실행으로 구조적 정합성 확인 가능.

---

### N3 — Image (Cut-off: 3)

| 원칙 | 적용 여부 | 판단 기준 |
|------|-----------|-----------|
| **계보적 불변성** | **핵심** | 부모 매스의 기하학적 형태가 렌더 이미지에서 왜곡 없이 유지되는가 |
| **결정론적 하드데이터** | **핵심** | GFX 100S 광학 파라미터 적용 + 물성 데이터(재료, 반사율) 규격화 여부 |
| 기능적 해결성 | 보조 | 시각적 실체화 JTBD 수행 여부 |

**Cut-off 3의 이유 (고엔트로피 처리):**  
이미지 생성은 AI 확률적 출력으로 단일 실행 시 광학/기하 안정성 보장이 어렵다. 3회 연속 성공을 요구함으로써 우연한 1회 통과를 배제하고 재현 가능한 품질 기준을 확보한다. 이는 오케스트레이터 프로토콜에서 "연속 성공 여부를 엄격히 따진다"고 명시한 유일한 노드이다.

---

### N4 — Viewpoint (Cut-off: 1)

| 원칙 | 적용 여부 | 판단 기준 |
|------|-----------|-----------|
| **계보적 불변성** | **핵심** | 시점 이동 전후 기하학적 형태 왜곡이 제로인가 |
| 결정론적 하드데이터 | 보조 | 시점 파라미터(방위각, 고도각, 원근법) 수치 기반 여부 |
| 기능적 해결성 | 보조 | 시점 이동 JTBD(새로운 공간 경험) 제공 여부 |

**Cut-off 1의 이유:** 시점 변경은 기하학 유지 여부만 검증하면 되는 단순 변환. 단일 확인으로 충분.

---

### N5 — Elevation (Cut-off: 2)

| 원칙 | 적용 여부 | 판단 기준 |
|------|-----------|-----------|
| 계보적 불변성 | 보조 | 이미지 원본의 입면 구조가 수치 데이터에 반영되었는가 |
| **결정론적 하드데이터** | **핵심** | AEPLSchema v4(폭/높이/깊이/공극비/재료) 수치 적합성 및 교차 검증 |
| 기능적 해결성 | 보조 | 시각 → 수치 역추출 JTBD 수행 여부 |

**Cut-off 2의 이유:**  
이미지에서 수치 데이터를 역추출하는 작업은 단일 실행으로 오차가 발생할 수 있다. 2회 교차 검증을 통해 AEPLSchema 수치의 내부 일관성(폭×층수=높이 등)을 확인한다.

---

### N6 — Diagram (Cut-off: 1)

| 원칙 | 적용 여부 | 판단 기준 |
|------|-----------|-----------|
| 계보적 불변성 | 보조 | 원본 도면/이미지의 공간 구조가 다이어그램에 반영되었는가 |
| 결정론적 하드데이터 | 보조 | 다이어그램 요소가 공간 데이터에 근거하는가 |
| **기능적 해결성** | **핵심** | 공간 로직이 의지적 내러티브(설계 의도 설명 가능한 다이어그램)로 변환되었는가 |

**Cut-off 1의 이유:** 다이어그램은 해석적 산출물로 단일 실행으로 JTBD 달성 여부 확인 가능.

---

### N7 — Print (Cut-off: 1)

| 원칙 | 적용 여부 | 판단 기준 |
|------|-----------|-----------|
| 계보적 불변성 | 보조 | 캔버스의 성과물이 레이아웃에 누락 없이 포함되었는가 |
| 결정론적 하드데이터 | 보조 | 인쇄 규격(DPI, 블리드, 여백) 준수 여부 |
| **기능적 해결성** | **핵심** | 전문가급 레이아웃 + 4개 이상의 타이포라인 계층 연결성 |

**Cut-off 1의 이유:** 최종 문서화 노드로 레이아웃 품질은 단일 실행으로 확인 가능.

## REALIZED 잠금 CONSTRAINT

```
Red Team FAIL (is_final_approved: false)
    → REALIZED 전환 절대 불가 (오케스트레이터 CONSTRAINT)
    → applyOrchestratorDecision() 에서 강제 차단
    → artboardStatus = 'VERIFYING' 유지

Red Team PASS (is_final_approved: true)
    → cumulativeScore + 1
    → IF cumulativeScore >= nodeCutoff → REALIZED
    → ELSE → VERIFYING (다음 실행 대기)
```

구현 위치: `project_canvas/lib/orchestrator.ts` → `applyOrchestratorDecision()`

## 원칙 위반 시 피드백 형식

Red Team은 FAIL 판정 시 오케스트레이터에게 아래 형식으로 결함을 보고한다.  
오케스트레이터는 이 피드백을 `transformation_logic`에 반영하여 재실행을 트리거한다.

```json
{
  "audit_report": {
    "node": "N3_Image",
    "constitution_check": {
      "Principle_1": 0,
      "Principle_2": 1,
      "Principle_3": 1
    },
    "cumulative_status": "1 / 3",
    "is_final_approved": false
  },
  "feedback_loop": {
    "retry_required": true,
    "critical_defect": "부모 매스의 코너 각도(87°)가 렌더 이미지에서 라운딩 처리됨 — 계보적 불변성 위반"
  }
}
```
