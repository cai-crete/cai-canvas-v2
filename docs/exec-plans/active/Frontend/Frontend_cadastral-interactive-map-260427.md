# Frontend Plan — 동적 지적도(Interactive Map) 및 썸네일 캡처 저장 시스템 구축

## 1. Goal Description
현재의 지적도 컴포넌트를 **완전한 동적 인터랙티브 맵(Pan, Zoom, Rotate 지원)**으로 전면 업그레이드합니다.
또한, 주변 필지 데이터를 병합하여 반경 500m 이내의 지적도를 한눈에 보여주며, 배경 지도 끄기(Layoff) 등 설정 기능은 우측 사이드바로 분리합니다.
특히 대표님께서 지적하신 **"지적도 노드 움찔거림(떨림) 현상"**을 완벽히 잡고, 최초 렌더링된 지적도 화면을 **"Base64 이미지 썸네일로 캡처하여 캔버스에 자동 저장"**하는 기능을 추가하여 최적의 성능을 달성합니다.

## 2. 사용자 피드백 반영 사항 및 답변

> [!IMPORTANT]
> **Q1. 썸네일 만이라도 저장하여 보여지게 해달라, 어려운가?**
> **A. 어렵지 않습니다. 완벽하게 가능합니다.**
> 지적도 폴리곤을 화면에 성공적으로 렌더링한 직후, 해당 화면(SVG 맵 전체)을 백그라운드에서 Canvas API를 이용해 Base64 이미지로 캡처(스냅샷)하겠습니다. 이 캡처된 이미지를 해당 노드의 `thumbnailData`에 자동 저장(Dispatch)하면, 다음번에 캔버스를 열 때 무거운 지적도 데이터를 다시 부르기 전이라도 이 스냅샷(썸네일 이미지)이 카드에 즉시, 예쁘게 뜨게 됩니다.
> 
> **Q2. 캔버스 상에서 움직이면 지적도 노드만 움찔거리는 현상이 있습니다. 썸네일을 이미지로 만들면 문제가 사라지는 것 아닌지?**
> **A. 대표님의 직관이 100% 정확합니다.**
> 현재 떨림 현상의 근본 원인은 캔버스를 줌/팬 할 때마다, '카드 내부'에 있는 수백 개의 지적선 SVG(라이브 컴포넌트)가 브라우저에 의해 실시간으로 크기가 재계산(리페인트)되기 때문입니다.
> 대표님 말씀대로, **카드 상태일 때는 무거운 라이브 맵 대신 '캡처된 썸네일 이미지 1장'만 딱 띄워버리면** 브라우저 연산이 0에 수렴하여 **움찔거림과 렉이 원천적으로 소멸**됩니다.
> 사용자가 지적도를 '확대(Expand)'하여 전체 화면으로 볼 때만 진짜 라이브 맵 엔진을 가동시키고, 캔버스 바닥에서는 오직 이미지만 보여주도록 아키텍처를 변경하겠습니다. 기술적 최적화에 매몰되어 본질을 놓쳤습니다. 정확한 지적이십니다.

## 3. Proposed Changes

### [MODIFY] `app/api/vworld-map/route.ts`
- **주변 지적도 병합**: 단일 PNU 조회에서 `BBOX` 필터 조회로 변경하여, 대상지 반경 500m 안의 모든 지적도 폴리곤을 한 번에 내려받도록 쿼리 수정.

### [MODIFY] `components/CadastralMapView.tsx`
- **비율 찌그러짐 및 움찔거림 방지**: 화면 창 비율(`preserveAspectRatio`)에 의존하지 않고, 위경도를 실제 Pixel 좌표로 변환하는 Web Mercator 투영 공식을 사용해 절대 크기로 렌더링. 부모 캔버스의 Transform 변경 시 불필요하게 리렌더링되지 않도록 철저히 격리.
- **Pan / Zoom / Rotate 엔진**: `onWheel`과 `onPointerMove` 이벤트를 이용해 SVG 캔버스 내부만 조작할 수 있는 변환(Transform) 행렬 상태 구현.
- **주변 필지 디자인**: 타겟 필지는 붉은색(면+선), 주변 필지는 옅은 회색 선으로 구분.
- **자동 캡처 엔진 (썸네일 저장)**: 렌더링 완료 후 `dom-to-image` 또는 네이티브 캔버스 변환을 통해 SVG를 스냅샷으로 찍고, `onThumbnailCaptured` 콜백을 통해 상위로 전달.

### [MODIFY] `app/page.tsx` & `components/NodeCard.tsx`
- **썸네일 저장 파이프라인**: `CadastralMapView`에서 올려보낸 스냅샷 이미지를 받아 해당 `cadastral` 노드의 `thumbnailData` 상태 업데이트.
- 노드 카드 로드 시 `thumbnailData`가 있으면 API 호출 여부와 상관없이 카드에 고해상도 이미지 썸네일을 즉각 렌더링.

### [NEW] `components/ExpandedSidebar/CadastralPanel.tsx`
- **지적도 전용 우측 탭**: 지적도 노드 확대 시 나타나는 전용 설정 패널.
- **기능**: 배경 타일 표시 유무(Layoff/Layon - None, Base, Satellite, Hybrid 라디오 버튼) 및 지도 회전(0~360도) 슬라이더.

### [MODIFY] `components/ExpandedSidebar/ExpandedSidebar.tsx`
- `node.type === 'cadastral'`일 경우 `CadastralPanel`이 렌더링되도록 탭 구조 통합.

## 4. Verification Plan
1. **썸네일 저장 검증**: 지적도 생성 후, 해당 노드 상태에 `thumbnailData`가 저장되는지 콘솔 및 리로드 후 확인.
2. **떨림(Jitter) 검증**: 캔버스 바닥을 잡고 마구 흔들고 줌인/아웃을 반복할 때 지적도 노드 자체가 깜빡이거나 사이즈가 진동하지 않는지 확인.
3. **찌그러짐(Distortion) 검증**: 창 크기를 가로/세로로 심하게 늘려도 지적도의 원래 가로세로 비율(정사각형 땅이 직사각형이 되지 않는지)이 완벽히 유지되는지 확인.
4. **우측 패널 조작**: 배경 타일을 "끄기(None)"로 설정 시 위성 사진이 사라지고 하얀 배경에 지적도 선만 남는지 확인.
