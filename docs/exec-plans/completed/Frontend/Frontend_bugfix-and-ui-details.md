## 목표

1. **자식 노드 생성 오류 수정**: 한 부모 노드에서 여러 자식 노드를 파생할 때 기존 노드가 교체되는(또는 같은 위치에 겹쳐 생성되어 교체되는 것처럼 보이는) 현상을 해결하고, `autoLayout.ts`의 `placeNewChild`를 통해 자식들이 나열되도록 합니다.
2. **엣지 연결 유지**: 노드 파생 시 부모와 자식 간의 연결선(Edge)이 정상적으로 생성되고 유지되도록 합니다.
3. **노드 복제 버그 수정**: 복제 시 아트보드의 썸네일과 속성 값들이 누락되어 빈 아트보드로 변하는 문제를 해결합니다.
4. **빈 아트보드 UI 개선**: 빈 아트보드도 다른 아트보드와 같은 테두리(stroke) 및 그림자 스타일을 적용하며, 내부의 '-' 표시를 제거하여 흰 화면으로 유지합니다.
5. **업로드 이미지 비율 유지**: 사용자가 업로드한 이미지가 아트보드 밖으로 잘리거나 깨지지 않고 원본 비율(`object-fit: contain`)을 유지하며, 빈 공간은 흰색으로 처리되도록 수정합니다.

## 제안하는 변경 사항

### project_canvas/app/page.tsx
- `placeNewChild` 모듈을 import 합니다 (`@/lib/autoLayout`).
- **자식 노드 생성 기반 마련**: 향후 API 연동 시 동일 부모에서 여러 자식이 파생되는 상황을 지원하기 위해, `selectedNodeId`가 있을 때 새로운 노드를 생성하면 `placeNewChild`를 통해 위치(position)를 계산하여 하단으로 나열(`pushdowns` 적용)되도록 `createAndExpandNode` 함수를 리팩토링합니다.
- 새 노드 생성 시 `parentId: selectedNodeId`, `autoPlaced: true`를 부여하고, 새로운 `CanvasEdge`를 생성하여 부모-자식 간의 `edges` 배열에 추가합니다.
- Expand가 불가능한 노드 탭(`elevation`, `viewpoint`, `diagram`)을 클릭했을 때 기존(부모) 노드를 교체하던 임시 로직을 수정합니다. 빈 아트보드가 아닌 경우 부모 노드는 소스로만 활용되고, 새로운 자식 노드(`createChildNode` 등의 로직)가 파생되어 캔버스에 추가되도록 통일합니다.
- `duplicateNode` 함수에서 복제 시 `hasThumbnail: false`로 덮어쓰던 부분을 제거하여 원본 속성(이미지 등)이 그대로 복제되도록 수정합니다.

### project_canvas/components/NodeCard.tsx
- `artboardBorder` 및 `artboardBoxShadow` 변수에서 `isBlank`에 따라 점선 및 다른 그림자를 적용하던 로직을 제거하고, 다른 아트보드와 동일한 스타일로 통일합니다.
- `isBlank` 조건부 렌더링 내부에 있던 `—` 표시 마크업을 제거하여 빈 하얀색 화면으로 유지되도록 합니다.
- `artboardType === 'image' && node.thumbnailData` 렌더링 부분의 `<img />` 스타일 중 `objectFit: 'cover'`를 `objectFit: 'contain'`으로 변경하여 비율이 깨지거나 잘리지 않게 합니다.
- 이미지가 있는 노드를 복제(Duplicate)했을 때, 복제된 노드에도 이미지가 정상적으로 보이는지 확인합니다.
- 빈 아트보드가 점선이 아닌 실선+그림자 디자인으로 노출되고 내부에 '-'가 없는지 확인합니다.
- 비율이 다른 이미지(가로로 길거나 세로로 긴 이미지)를 업로드했을 때 이미지가 잘리지 않고 정상 비율로 레터박스가 생기는지 확인합니다.
