# 작업지시서: Sketch-to-Plan 지적도 미표현 / 그리드 출력 오류 수정

## 배경
- 이전 수정(260503)에서 외벽 왜곡 방지를 위해 Phase 2에서 지적도를 완전 제거함
- 결과: 생성 평면도가 흰 배경의 순수 건축 도면만 출력 → 지적도 컨텍스트 소멸
- 또한 exportStrokesOnly()에 그리드를 bake-in하여 AI가 출력에 그리드를 재현함
- 원칙: 지적도 위에 스케치할 때 지적도는 단일 이미지 안에 반드시 표현되어야 함

## 수정 파일
- `project_canvas/components/SketchCanvas.tsx`
- `render-server/src/routes/sketchToPlan.ts`

## 근거 (판단 기록)
- Phase 1 실패(외벽 혼동)는 "어떤 경계를 외벽으로 선택할지" 판단 과정의 실패
- Phase 2는 spatial-spec이 building_footprint 좌표를 명시적으로 제공 → AI가 선택할 필요 없음
- 따라서 Phase 2에서 지적도를 재투입해도 외벽 혼동 위험은 Phase 1보다 구조적으로 낮음
- 지적도 없는 경우(단독 스케치)는 기존 흰 배경 동작 유지

- [ ] 생성 이미지에 그리드 선이 나타나지 않는지
- [ ] 지적도 없을 때: 흰 배경 평면도가 정상 생성되는지
