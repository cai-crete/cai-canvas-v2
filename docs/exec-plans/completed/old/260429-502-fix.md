**요청자**: bzoo@cre-te.com  
**증상**: `https://cai-canvas-v2.vercel.app/` → 502 에러  
**정상 URL**: `https://cai-canvas-v2-8ajl2ixcr-cai-cretes-projects.vercel.app/` → 정상 작동

## 원인 분석

### 원인 A — `RenderWakeup` 컴포넌트가 마운트되지 않음 (코드 버그) ★ 확실

| 파일 | 상태 |
|------|------|
| `project_canvas/components/RenderWakeup.tsx` | 파일 존재, 로직 정상 |
| `project_canvas/app/layout.tsx` | `RenderWakeup` import **없음** |
| `project_canvas/app/page.tsx` | `RenderWakeup` import **없음** |

**결과**: Render 서버(AI + Print)가 절대 pre-warm되지 않음.  
Render 무료 티어는 15분 비활성 후 슬립 → 첫 요청에 ~30초 지연.  
Vercel 서버리스 함수가 Render 응답 대기 중 연결 실패 → **502**.

---

### 원인 B — Vercel 환경변수가 Production에 미설정 가능성 ★ 의심

Vercel에서 env var 추가 시 체크박스를 선택:
- `[ ] Production`  `[x] Preview`  `[x] Development`

Preview URL은 정상 작동 → Preview 환경 env vars 정상.  
Production URL 502 → **Production 체크박스 누락 가능성**.

**특히 위험한 변수**:
- `NEXT_PUBLIC_RENDER_HEALTH_URL` — 빌드 타임에 JS 번들에 삽입됨.  
  Production에 미설정 시 클라이언트 번들에 `undefined` 포함 → 모든 Render 호출 실패.
- `RENDER_SERVER_URL` + `INTERNAL_SECRET` — 미설정 시 모든 AI 라우트 500.

---

### 원인 C — Vercel `/api/print-proxy/*` 라우트의 env vars 미문서화 (잠재적 문제)

```
project_canvas/app/api/print-proxy/api/print/route.ts   → PRINT_API_URL, CANVAS_API_SECRET 필요
project_canvas/app/api/print-proxy/api/library/route.ts → PRINT_API_URL, CANVAS_API_SECRET 필요
project_canvas/app/api/print-proxy/[...path]/route.ts   → PRINT_API_URL, CANVAS_API_SECRET 필요
```

현재 클라이언트(`ExpandedView.tsx`)는 Render를 직접 호출하므로 이 라우트들은 **현재 미사용**.  
그러나 README_server.md에 Vercel env vars로 미기재 → 추후 혼란 야기 가능.

**파일**: `project_canvas/app/layout.tsx`


### Phase 2: Vercel 대시보드 — env vars Production 체크 확인

Vercel Dashboard → cai-canvas-v2 project → Settings → Environment Variables

아래 변수들이 **Production 환경에도 체크**되어 있는지 확인:

| 변수명 | Production 필수 | 비고 |
|--------|----------------|------|
| `RENDER_SERVER_URL` | ✅ 필수 | AI 라우트 서버사이드 |
| `INTERNAL_SECRET` | ✅ 필수 | AI 라우트 인증 |
| `NEXT_PUBLIC_RENDER_HEALTH_URL` | ✅ 필수 | 클라이언트 번들 삽입 |

**미설정 항목 발견 시**:
1. 변수 수정 → Production 체크박스 추가
2. Vercel Dashboard → Deployments → 최신 배포 → **Redeploy** (Rebuild)

> `NEXT_PUBLIC_*` 변수는 값 변경 후 Redeploy를 **반드시** 해야 번들에 반영됨.

- [ ] Vercel에서 3개 변수 모두 Production 체크 확인
- [ ] 미설정 항목 수정 후 Redeploy 실행
