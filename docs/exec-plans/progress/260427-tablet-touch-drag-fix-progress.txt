## 방금 한 일

태블릿(iPad Safari)에서 ExpandedView 스케치 시 웹 페이지가 드래그되는 현상 수정.

## 원인

Safari on iPadOS에서 네이티브 제스처 인식기는 JS 포인터 이벤트보다 먼저 실행됨.
`<canvas>` 요소에는 `touchAction: 'none'`이 있었지만, 이를 감싸는 컨테이너 div들과
툴바/사이드바 영역에는 없었음. 캔버스 밖 영역에서 터치가 시작되면 브라우저가
스크롤 제스처를 시작하고, 펜이 캔버스로 이동해도 제스처가 지속됨.

## 변경 파일

1. project_canvas/app/globals.css
   - `html, body`에 `touch-action: none` 추가 (전역 bounce/overscroll 차단)

2. project_canvas/components/SketchCanvas.tsx
   - containerRef div에 `touchAction: 'none'` 추가

3. project_canvas/sketch-to-image/ExpandedView.tsx
   - 외부 wrapper div에 `touchAction: 'none'` 추가

4. project_canvas/sketch-to-plan/ExpandedView.tsx
   - 외부 wrapper div에 `touchAction: 'none'` 추가

- 작업지시서 completed 폴더로 이동 완료
