# 상세 변경 코드 내용

## 1. Print_ExpandedView.tsx
- UI State 추가: `const [infoToastMsg, setInfoToastMsg] = useState<string | null>(null)`
- Toast Timer 로직 추가: `infoToastMsg` 발생 시 3초 후 `null` 로 자동 소멸되도록 `useEffect` 적용.
- Initial Action 할당 보완 (`processImages` 훅):
  `initialAction === 'video'` 상태로 진입 시, `newFiles` 길이가 0 초과라면 `videoStartImage`, `videoEndImage`에 바로 할당 및 이미지가 2장을 초과할 시 `infoToastMsg` 설정.
- 모드 변경 로직 (`handleModeChange`):
  - **역방향 복구**: VIDEO -> REPORT/PANEL/DRAWING 모드로 변경 시 `videoStartImage`/`videoEndImage`에 있는 이미지를 다시 `images` 갤러리 배열로 복구.
  - **정방향 할당**: 다른 모드에서 VIDEO로 변경될 때 `images`에 있던 첫 2장을 Start/End 슬롯에 할당하고, 2장 초과 시 "비디오 생성에는 2장의 이미지만 사용할 수 있습니다." Toast 표시.
- InfoToast UI 마크업 추가: `return` 구문 최상단에 `GeneratingToast`와 동일한 룩앤필(Pill 형태, 하단 중앙 플로팅 32px, Pretendard 폰트, Shadow)을 가진 Toast View 마크업 구현.

## 2. route.ts (API)
- `fal.storage.upload` 개별 에러 로깅 (`Promise.all` 내 처리):
  각 `upload(blob)` 호출에 대해 개별적인 `.catch` 구문을 연결하여 어느 이미지가 업로드에 실패했는지 명확히 알 수 있도록 Error 메시지 세분화 ("Start 이미지 업로드 실패: ...", "End 이미지 업로드 실패: ...").
- 반환값 예외 방어:
  fal.ai 클라이언트 버전 차이에 따라 upload 반환값이 문자열(string) 또는 객체(`{ url: string }`)로 다를 수 있음에 대비하여 `typeof` 조건식을 통한 하위 호환성 확보.
- 결과 추출 예외 방어:
  비디오 결과인 `result` 객체 구조 차이(예: `result.data.video.url` vs `result.video_url`)를 감안해 옵셔널 체이닝 및 null 병합 연산자(`??`)를 통한 방어 코드 적용.
