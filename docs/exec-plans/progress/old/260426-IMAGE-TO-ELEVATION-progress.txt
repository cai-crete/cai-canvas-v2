### 분석한 내용

**ELEVATION 레포 핵심:**
- Protocol A: Gemini Vision → AEPLSchema (건물 치수·재료·분절 분석)
- Protocol B: 5방향 입면도 병렬 생성 (front/rear/left/right/top)
- front = 원본 이미지 passthrough, 나머지 4개 AI 생성
- 모델: 분석 gemini-3.1-pro-preview, 이미지 gemini-3.1-flash-image-preview
- 폴백: gemini-2.5-pro-preview / gemini-2.5-flash-image
- CrossGrid 5-view 십자형 레이아웃

**cai-canvas-v2 현황:**
- NodeType 'elevation' 이미 types/canvas.ts에 정의됨
- ARTBOARD_COMPATIBLE_NODES.image에 포함
- 그러나 NODES_NAVIGATE_DISABLED에 있어 ExpandedView 없이 자식 노드만 생성됨
- project_canvas/elevation/ 폴더 없음

### 확정된 노드 흐름
1. 이미지 아트보드 → ELEVATION 탭 클릭 → 입력 노드 A 생성 + 즉시 expand
2. ExpandedView: 원본 이미지 표시 + PROMPT + GENERATE
3. GENERATE → 즉시 캔버스 복귀 + "ELEVATION GENERATING" toast
4. 백그라운드 API → 완료 시 결과 노드 B 생성
5. 결과 노드 B: CrossGrid 합성 이미지를 thumbnailData로 표시
6. 결과 노드 B expand → CrossGrid 전체 + EXPORT 버튼

### 확정된 CTA 버튼 동작
- 생성 전: GENERATE (검정 pill)
- 생성 중: GENERATING... + 스피너 (disabled)
- 생성 후: EXPORT (아웃라인 pill, 5개 이미지 일괄 다운로드)
- PROMPT 변경 시 EXPORT → GENERATE로 리셋
