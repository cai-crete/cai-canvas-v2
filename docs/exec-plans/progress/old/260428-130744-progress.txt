[Fix 4] GENERATE 413 Payload Too Large 오류
  - 증상: GENERATE 클릭 시 사이드바에 "Unexpected token 'R', 'Request En'... is not valid JSON" 표시
  - 콘솔: :3900/api/print-proxy/api/print → 413 (Payload Too Large)
  - 원인 분석:
    · Next.js 15.1+ 에서 Route Handler도 기본 1MB body size limit 적용됨
    · print 서버(3777): next.config.ts에 bodySizeLimit: '50mb' 이미 설정됨 ✅
    · Canvas BFF(3900): next.config.ts에 bodySizeLimit 미설정 → 기본 1MB 적용 ❌
    · 이미지 파일이 1MB 초과 시 Next.js가 route handler 실행 전에 413 plain-text 반환
    · 프론트엔드 res.json()이 plain-text 413 파싱 실패 → JSON parse error
  - 수정: project_canvas/next.config.ts에 experimental.serverActions.bodySizeLimit: '50mb' 추가
  - 파일: project_canvas/next.config.ts

[Fix 5] ERR_INVALID_URL — selectedImages base64 줄바꿈 문자
  - 증상: 콘솔에 "Failed to load data:image/png;base64--... ERR_INVALID_URL" 두 건
  - 원인: parseNodeImage()가 node.generatedImageData에서 base64 추출 시 줄바꿈(\n) 제거 안 함
           → Print_ExpandedView.tsx가 fetch(`data:${mimeType};base64,${base64}`) 호출 시 URL 무효
  - 수정: parseNodeImage() 반환 시 current.replace(/\s/g, '') 추가
  - 파일: project_canvas/print/ExpandedView.tsx
