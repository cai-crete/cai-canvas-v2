## 완료된 작업

### [BUG-1] "Cannot redefine property: viewer" 재진입 에러 — 근본 수정

**이전 시도의 실패 원인:**
- per-node 싱글톤 레지스트리: Fast Refresh 시 JS 모듈이 리셋 → _registry Map 초기화 → initialized=false → 재시도 → 에러
- mountKey React state: unmount 시 setState는 마운트 전에는 무효 → 항상 동일 ID → 미해결
- 두 useEffect([containerId], [heading])에서 동시에 tryInit 호출 → race condition

**근본 해결책:**
1. 전역 단일 div (id="vw3d-global-singleton") — 페이지에 1개만 존재
2. 초기화 상태를 div.dataset.initialized='true'에 저장 → DOM은 Fast Refresh 후에도 유지됨
3. map.start() 호출 전 동기 플래그 세팅 → race condition 차단
4. tryInit을 heading useEffect에서만 호출 (이전: 2곳에서 동시 호출 가능)
5. 이미 초기화된 경우 → moveCamera()만 호출 (Cesium camera.setView or vw.setCamera)

**수정 파일:** project_canvas/components/Map3DView.tsx (전면 재작성)

## 주의사항
- Fast Refresh 문제: 개발 서버에서 파일 저장 시 JS 모듈 리셋 → _map 변수 null → 
  isDivInitialized()가 true이면 카메라만 이동 시도하나, _map이 null이면 moveCamera 무동작

  → 프로덕션 빌드에서는 이 문제 없음
