## 방금 무엇을 했는지

법규 MCP가 동작하지 않는 원인 분석 및 수정.

### 근본 원인 (Root Cause)

**api/generate.ts**에서 `tools` 파라미터가 `@google/genai` SDK의 `generateContent()` 최상위 레벨에
스프레드되어 있었음. SDK는 `tools`를 `config` 내부에서만 인식하므로 Gemini에게 MCP 도구가
전혀 전달되지 않았음.

```typescript
// 수정 전 (버그):
ai.models.generateContent({
  model, contents, config,
  ...(tools ? { tools } : {}),  // ← SDK가 무시! Gemini는 tools를 받지 못함
});

// 수정 후:
ai.models.generateContent({
  model, contents,
  config: { ...config, ...(tools ? { tools } : {}) },  // ← config 내부에 병합
});
```

**결과**: Gemini가 Function Call을 전혀 요청하지 않음 → Agent Loop가 매번 첫 번째 반복에서
text 응답으로 종료 → MCP 도구가 한 번도 호출되지 않음.

### 수정한 파일

1. **api/generate.ts** (핵심 버그 수정)
   - `tools`를 `config` 내부에 병합하여 Gemini에게 올바르게 전달
   - `http://localhost:5174` (APP_URL) CORS 허용 목록 추가
   - `https://cai-planners-v2.vercel.app` CORS 추가

2. **api/mcp-proxy.ts** (방어적 보강)
   - MCP 서버가 SSE 형식으로 응답하는 경우를 대비한 파싱 로직 추가
   - ALLOWED_ORIGINS 상수로 정리 + localhost:5174 추가

### 확인 사항

- `node test-mcp.mjs` → MCP 서버 자체는 정상 (16개 도구, search_law 작동)
- `npx tsc --noEmit` → 타입 오류 없음
